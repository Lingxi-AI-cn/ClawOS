### 2. 云端模型 OAuth 认证（Google Antigravity）

**设计原则**: Prod 模式下不内置任何预设凭据，用户必须提供自己的 OAuth 应用凭据。

**新增向导步骤** (`oauth-creds`)
- 引导用户在 Google Cloud Console 创建 OAuth 2.0 客户端
- 将 `clawos://oauth-callback` 添加为授权重定向 URI
- 用户填入 Client ID + Client Secret 后进入授权流程

**Android 原生实现** (`ClawOSBridge.java`)
- PKCE 流程（`code_verifier` + `code_challenge` SHA-256）
- 打开系统浏览器完成 Google 授权
- 通过 Deep Link (`clawos://oauth-callback`) 回调
- 用用户提供的 Client ID/Secret 换取 access_token + refresh_token
- 自动获取 Cloud Code Assist `projectId`

**AndroidManifest.xml**
```xml
<intent-filter>
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="clawos" android:host="oauth-callback" />
</intent-filter>
```

**MainActivity.java** — 新增 `onNewIntent` 处理 OAuth 回调：
```java
@Override
protected void onNewIntent(Intent intent) {
    super.onNewIntent(intent);
    Uri data = intent.getData();
    if (data != null && "clawos".equals(data.getScheme()) && "oauth-callback".equals(data.getHost())) {
        ClawOSBridge.handleOAuthCallback(data);
    }
}
```

**文件**: `ui/src/components/AddProviderWizard.tsx`, `ui/src/gateway/bridge.ts`,
`ui/android/app/src/main/java/com/clawos/plugins/ClawOSBridge.java`,
`ui/android/app/src/main/java/com/clawos/app/MainActivity.java`,
`ui/android/app/src/main/AndroidManifest.xml`

## ROM 构建过程中发现的问题与修复

### 问题 1: TypeScript 编译错误
**文件**: `ui/src/components/ModelSelector.tsx:453`
**错误**: `onClick={onAddProvider}` — 函数签名不匹配 `MouseEventHandler`

### 问题 2: Java 编译错误
**文件**: `ui/android/app/src/main/java/com/clawos/app/MainActivity.java`
**错误**: `cannot find symbol: class Uri`
**修复**: 添加 `import android.net.Uri;`

### 问题 3: 模拟器卡在 "phone is starting"
**根本原因**: 构建流程有三处错误：
1. 使用了 `assembleRelease`（无签名APK），应使用 `assembleDebug`（有 debug 签名）
2. 没有运行 `05-setup-device-tree.sh` 同步设备树到 `/opt/aosp`
3. 增量构建使用了 `out/` 中的旧 APK 缓存（MD5 不一致）

**修复步骤**:
```bash
# 1. 构建 debug APK（必须用 debug，不能用 release）
export JAVA_HOME=$JAVA_HOME  # 设置 JDK 路径，参见 env.example
cd $CLAWOS_ROOT/ui/android
./gradlew assembleDebug
cp app/build/outputs/apk/debug/app-debug.apk ../../aosp/device/clawos/apps/ClawOS.apk

# 2. 同步设备树（每次修改 aosp/device/clawos/ 后必须执行）
cd /opt/ClawOS
bash aosp/scripts/05-setup-device-tree.sh

# 3. 删除旧 APK 缓存（增量构建不会自动更新 PRESIGNED APK）
rm /opt/aosp/out/target/product/emulator_arm64/product/priv-app/ClawOS/ClawOS.apk
rm -rf /opt/aosp/out/target/product/emulator_arm64/product/priv-app/ClawOS/oat/

# 4. 重新构建
cd /opt/aosp
source build/envsetup.sh && lunch clawos_arm64-userdebug
make -j$(nproc)
make emu_img_zip -j$(nproc)
```

## 正确的完整构建流程（总结）

参考 `.cursor/rules/clawos-project.mdc` 中的"ROM 构建完整指南"：

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1a | `cd ui && pnpm run build` | 构建 React UI |
| 1b | `npx cap sync android` | 同步 Web 资产到 Android |
| 1c | `JAVA_HOME=... ./gradlew assembleDebug` | 构建 debug APK |
| 1d | `cp app-debug.apk ../../aosp/device/clawos/apps/ClawOS.apk` | 复制 APK |
| 2 | `bash aosp/scripts/05-setup-device-tree.sh` | 同步设备树 |
| 3 | `source envsetup.sh && lunch ... && make -j$(nproc)` | AOSP 编译 |
| 4 | `make emu_img_zip -j$(nproc)` | 打包镜像 |
| 5 | Mac: `bash aosp/run-emulator-mac.sh --pull --lan --clean` | 拉取并启动 |

⚠️ **关键注意事项**:
- 必须用 `assembleDebug`，不能用 `assembleRelease`（PRESIGNED APK 需要有签名）
- 每次修改设备树后必须运行 `05-setup-device-tree.sh`
- 更新 ROM 后 Mac 端必须加 `--clean` 标志清除 AVD 缓存
- 增量构建可能不更新 PRESIGNED APK，需手动删除 `out/` 中的旧 APK
